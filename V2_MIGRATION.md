# AKFES v2 Migration

이 브랜치는 AKFES를 Tauri v2 기반 데스크톱 애플리케이션으로 리팩터링하는 작업 공간입니다.

- Desktop: Tauri v2 + Rust + React + TypeScript
- Server: FastAPI + Uvicorn
- Hardware: Arduino UNO + USB Serial
- Security target: 서버 중심 권한 검증, 장치 바인딩, 일회용 챌린지, 요청 서명, AES-256-GCM

## 장치 바인딩 정책

`AKFES_DEVICE_BINDING_REQUIRED=true`가 기본값입니다.

1. 로그인 요청에는 `device_id`가 반드시 필요합니다.
2. 라이선스의 첫 정상 로그인 장치가 해당 라이선스에 바인딩됩니다.
3. 이후 다른 장치 ID로는 같은 라이선스에 로그인할 수 없습니다.
4. 세션·챌린지·서명 요청에도 동일한 `X-AKFES-Device-ID`가 필요합니다.
5. 관리자가 바인딩을 초기화하면 기존 세션은 즉시 무효화됩니다.
6. 초기화 이후 다음 정상 로그인 장치가 새 장치로 바인딩됩니다.

장치 ID 원문은 라이선스 레코드에 저장하지 않고 HMAC-SHA256 다이제스트만 저장합니다.

## 요청 서명 규칙

```text
AKFES-V2
HTTP_METHOD
REQUEST_PATH
CHALLENGE
SHA256_REQUEST_BODY
DEVICE_ID_OR_EMPTY
```

클라이언트는 세션 토큰을 HMAC 키로 사용해 SHA-256 서명을 계산합니다. 챌린지는 한 번만 사용할 수 있고, 발급받은 세션과 다른 세션에서는 사용할 수 없습니다.

## 검증 상태

- 첫 장치 자동 바인딩 테스트 추가
- 장치 ID 누락과 다른 장치 로그인 거부 테스트 추가
- 관리자 초기화 후 기존 세션 무효화 테스트 추가
- 새 장치 재바인딩과 감사 로그 테스트 추가
- 정상 서명·본문 변조·챌린지 재사용 거부 테스트 유지
- 실제 Arduino 연결과 Windows Tauri 빌드는 해당 개발 환경에서 추가 검증 필요

## 현재 한계

서버 인증·장치 바인딩·요청 서명 계층은 준비됐지만 파일 암호화·복호화 API와 Tauri 클라이언트는 아직 이 계층을 사용하지 않습니다.
